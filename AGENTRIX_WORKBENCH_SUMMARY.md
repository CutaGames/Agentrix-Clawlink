# Agentrix 工作台总结 (Agentrix Workbench Summary)

## 1. 核心愿景 (Core Vision)
Agentrix 工作台旨在为 AI 代理生态系统提供一个“Agent First”的闭环环境。它通过消除商户、开发者和个人用户之间的功能断裂，实现了从开发到支付再到审计的全链路支持。

## 2. 三大核心路径 (Three Core Paths)

### Path 1: 商户上线 (Merchant Go-live)
*   **功能**: 提供了 100% 进度的引导式清单。
*   **关键步骤**:
    *   身份验证 (Identity Verification)
    *   商品同步 (Product Integration) - 支持 Shopify/WooCommerce。
    *   沙盒模拟 (Sandbox Testing)
    *   合规证明导出 (Proof Bundle) - 生成加密签名的审计报告。

### Path 2: 开发者构建 (Developer Build & Pack)
*   **功能**: 技能全生命周期管理。
*   **关键特性**:
    *   **Skill Registry**: 结构化定义 Agent 技能。
    *   **Multi-Packer**: 一键将技能转换为 **OpenAI Actions (OpenAPI)**、**Claude MCP** 或 **Gemini** 格式。
    *   **Test Harness**: 在发布前进行 E2E 逻辑验证。

### Path 3: 个人授权与审计 (Personal Security & Audit)
*   **功能**: 基于回执的透明审计系统。
*   **关键特性**:
    *   **Agent Authorizations**: 管理 Agent 的单次支付额度或最大回撤限制。
    *   **Behavior Audit**: 将复杂的链上/API 操作转化为**人类可读 (Human-readable)** 的自然语言回执。
    *   **MPC Security**: 集成 MPC 安全设置，确保私钥安全。

## 3. 技术架构 (Technical Architecture)

### 前端 (Frontend)
*   **框架**: Next.js (Pages Router)
*   **主入口**: `/agent-enhanced` (Unified Workbench)
*   **组件库**: Lucide-react, Tailwind CSS.
*   **角色切换**: 通过 `RoleSwitcher` 实现 Personal / Merchant / Developer 模式切换。


### 后端 (Backend - Skill Module)
*   **框架**: NestJS
*   **核心逻辑**: `SkillConverterService` 负责多生态 Schema 转换。
*   **审计系统**: 通过 `receipt_id` 贯通前后端，实现全链路追踪。

*   一套标准化的 **Skill-to-Ecosystem** 转换逻辑。
