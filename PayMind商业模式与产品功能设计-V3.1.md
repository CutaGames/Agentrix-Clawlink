## 3. 分成机制设计

### 3.1 交易分成机制（核心）

#### 3.1.1 基础分成结构

**总交易金额 = 商户收入 + Agent 分成 + PayMind 分成 + Provider 手续费**

#### 3.1.2 分成比例（按订单类型）

| 订单类型 | 商户收入 | Agent 分成 | PayMind 分成 | Provider 手续费 | 说明 |
|---------|---------|-----------|-------------|----------------|------|
| **实物商品** | 97.5% | 2% | 0.5% | 0% | 无 Provider |
| **实物商品（法币转数字货币）** | 94.5% | 2% | 0.5% | 3% | 有 Provider |
| **服务/数字资产** | 96% | 3% | 1% | 0% | 无 Provider |
| **服务/数字资产（法币转数字货币）** | 93% | 3% | 1% | 3% | 有 Provider |
| **NFT/虚拟资产** | 96% | 3% | 1% | 0% | 无 Provider |
| **NFT/虚拟资产（法币转数字货币）** | 93% | 3% | 1% | 3% | 有 Provider |

**说明**：
- **Agent 分成**：只有通过 Agent 推荐/执行的交易才有，直接购买无 Agent 分成
- **PayMind 分成**：所有交易都有
- **Provider 手续费**：只有法币转数字货币时才有

#### 3.1.3 Agent 分成分配（多 Agent 协作）

当多个 Agent 参与交易时，分成分配规则：

**场景 1：单一 Agent 推荐**
- 该 Agent 获得 100% 的 Agent 分成

**场景 2：多个 Agent 协作（推荐链）**
- **推荐 Agent（Referral Agent）**：30% 的 Agent 分成
- **执行 Agent（Execution Agent）**：70% 的 Agent 分成

**示例**：
- Agent A 推荐商户，Agent B 执行交易
- 交易金额：1000 USDC
- Agent 总分成：1000 × 3% = 30 USDC
- Agent A（推荐）：30 × 30% = 9 USDC
- Agent B（执行）：30 × 70% = 21 USDC

**场景 3：Agent 推荐商品（非商户）**
- 商品本身有佣金率（商户设置）
- Agent 获得商品佣金率的分成
- PayMind 获得平台手续费（0.5% 或 1%）

### 3.2 商户推广分成机制（关键创新）

#### 3.2.1 推广奖励结构（长期化）

**目标**：让接入 SDK 的 Agent 主动帮 PayMind 找商户，并与商户长期共成长

**奖励机制**：

- **长期交易分成**（唯一激励）
  - Agent 推荐的商户，其所有交易额的 **0.5%** 作为长期分成
  - 分成期限：**永久**（只要商户在平台）
  - 分成对象：推荐 Agent
  - 体现“持续服务 = 持续收益”，避免一次性补贴带来的短期行为

**示例**：
- Agent A 推荐商户 B 入驻
- 商户 B 月交易额：$100,000
- Agent A 月分成：$100,000 × 0.5% = **$500/月**
- 如果商户 B 持续运营 12 个月，Agent A 累计获得：**$6,000**
- 商户体量越大，Agent 分成越高，激励 Agent 提供长期服务

#### 3.2.2 推广 Agent 资格

**谁可以成为推广 Agent？**
- ✅ 任何接入 PayMind SDK 的 Agent
- ✅ 任何在 Marketplace 上架的 Agent
- ✅ 任何 PayMind 用户（通过快速生成 Agent 功能）

**推广流程**：
1. Agent 识别潜在商户
2. Agent 引导商户注册 PayMind
3. Agent 帮助商户完成入驻流程
4. 商户完成首次交易
5. Agent 获得奖励和长期分成

- **收益示例**：月 GMV 100K 美元的商户，可为推广 Agent 带来 $500/月 的持续收益

**推广流程**：
1. Agent 识别潜在商户

3. Agent 帮助商户完成入驻
4. 商户开始交易

#### 6.1.2 用户快速生成 Agent

**目标**：让所有用户都能在 5 分钟内生成自己的 Agent

**实现方式**：
1. 模板化生成（零代码）
2. 对话式生成（AI 辅助）

**Agent 类型**：
- 购物 Agent
- 空投 Agent
- 套利 Agent
- 返现 Agent
- Launchpad Agent
- 交易 Agent

### 6.2 Marketplace 资产扩张策略（立即生效）

**目标**：无需等待商户入驻，也能在 1 天内让 Marketplace 拥有成千上万的可交易资产，并持续指数级增长。策略分三阶段执行：**

#### 第一阶段：极速聚合（0-1 周）——“把所有链上资产都先变成商品”

| 方案 | 聚合内容 | Agent 能力 |
|------|----------|-----------|
| **Token Directory** | Solana token-list、Ethereum ERC20 list、BSC/Polygon/Sui/TON tokenlists | 自动分类、即刻支持 swap / buy / sell / limit-buy |
| **DEX 交易对** | Jupiter/Raydium、Uniswap/1inch、OpenOcean/LI.FI | 交易对=商品，支持市价单、限价单、定投、反向网格、AI 调参 |
| **NFT Collections** | Magic Eden、OpenSea、Tensor Trending | 支持买入、出价、扫地、挂单 |
| **链上 RWA** | USYC、ONDO、MANTRA、Maple、Credix 等 | 显示年化、购买入口、自动分红监控，无需项目方入驻 |
| **Launchpad/Presale** | Pump.fun、Raydium AcceleRaytor、TON Presale 等 | Agent 自动提醒、抢购、止盈、社交跟单 |

> 每一个 token / 交易对 / NFT / RWA / Launchpad 项目都等同于一个“商品”，Marketplace 在一天之内即可拥有几千到几万个可交易条目。

#### 第二阶段：半自动商户入驻（1-2 周）——“让项目方愿意进来”

| 路径 | 描述 | 收益机制 |
|------|------|----------|
| **开放上架入口（表单/API）** | 上传 logo、简介、交易入口、返佣配置 | 项目方设置 3-10% 返佣，PayMind 提供流量 |
| **Referral SDK** | 项目方在白皮书/推文中嵌入 PayMind 链接 | Agent 自动获得导流和分成 |
| **Agent 自助上架** | “谁上架，谁获得分成”，PayMind 审核合法性 | 激励 Agent 主动扩充资产池 |
| **开发者共建** | 提供 SDK/API，允许开发者抓取数据、添加新资产 | 加入 Marketplace 可获上架奖励 |

#### 第三阶段：AI 自动化扩张（1-2 月）——“AI 自己把资产池做大”

AI 将具备以下能力：

- 扫描 trending tokens / collections / pairs
- 检查链上指标（流动性、交易量、社交热度）
- 自动生成项目介绍、风险评级、交易入口
- 自动配置策略（定投、止盈、跟单、自动套利）
- 自动决定上架/下架/权重，保持资产最新

> 终局：PayMind 成为“AI 驱动的 Crypto Marketplace”，资产池持续领先竞争对手，Agent “边发现边上架”，形成资产自增长飞轮。

## 7. 分成机制详细设计

### 7.1 交易分成详细规则

#### 7.1.1 基础分成（无推广 Agent）

**场景**：用户通过 Agent A 购买商户 B 的商品

**分成计算**：
- 交易金额：$1000
- Agent A 分成：$1000 × 3% = $30（服务/数字资产）
- PayMind 分成：$1000 × 1% = $10
- 商户收入：$1000 - $30 - $10 = $960

#### 7.1.2 推广 Agent 分成（有推广 Agent）

**场景**：Agent C 推荐了商户 B，用户通过 Agent A 购买商户 B 的商品

**分成计算**：
- 交易金额：$1000
- Agent A（执行）分成：$1000 × 3% × 70% = $21
- Agent C（推荐商户）分成：$1000 × 0.5% = $5（长期分成）
- PayMind 分成：$1000 × 1% = $10
- 商户收入：$1000 - $21 - $5 - $10 = $964

**说明**：
- Agent C 的 0.5% 是长期分成，不占用 Agent 分成池
- Agent A 的 70% 是执行分成，还有 30% 给推荐 Agent（如果有）

#### 7.1.3 多 Agent 协作分成

**场景**：Agent D 推荐商品，Agent A 执行交易

**分成计算**：
- 交易金额：$1000
- 商品佣金率：10%（商户设置）
- Agent D（推荐商品）分成：$1000 × 10% × 30% = $30
- Agent A（执行）分成：$1000 × 10% × 70% = $70
- PayMind 分成：$1000 × 1% = $10
- 商户收入：$1000 - $100 - $10 = $890

---

### 7.2 推广分成详细规则

#### 7.2.1 商户推广奖励（长期化）

| 项目 | 说明 |
|------|------|
| **分成比例** | 商户交易额的 **0.5%** |
| **分成期限** | 永久（只要商户在平台） |
| **结算周期** | 每周（T+7） |
| **结算币种** | 默认 USDC/USDT，可选法币 |

**收益示例**：
- Agent E 推荐中型商户 F（月 GMV：$50,000）
- Agent E 月长期分成：$50,000 × 0.5% = $250
- Agent E 年长期分成：$250 × 12 = $3,000
- 与商户发展绑定，Agent 将持续提供服务以获取长期收益

---

### 7.3 分成结算机制

#### 7.3.1 结算周期

- **Agent 交易分成**：T+1 结算（交易完成后 1 天）
- **推广长期分成**：T+7 结算（每周结算一次）

#### 7.3.2 结算方式

- **数字货币**：USDC/USDT（优先）
- **法币**：通过 Provider 转换为法币（可选）
