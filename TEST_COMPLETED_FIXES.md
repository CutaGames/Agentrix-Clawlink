## 🧪 测试步骤

### 测试1: 首页查看演示按钮

1. **访问首页**
   ```
   http://localhost:3000
   或
   http://<WSL2_IP>:3000
   ```

2. **测试查看演示按钮**
   - 在Hero区域找到"查看演示"按钮
   - 点击按钮
   - **预期结果**: 应该跳转到 `/use-cases` 页面（应用场景页）

3. **验证跳转**
   - 确认URL变为 `http://localhost:3000/use-cases`
   - 确认页面显示应用场景内容

### 测试2: Footer组件

1. **滚动到页面底部**
   - 在首页或其他页面滚动到底部

2. **检查Footer内容**
   - **社交账号图标**: 应该看到X/Twitter、Telegram、Discord三个图标
   - **产品链接**: 应该看到API文档、SDK文档、应用场景、功能特性
   - **开发者资源**: 应该看到快速开始、GitHub、集成指南、示例代码
   - **法律信息**: 应该看到隐私政策、服务条款、免责声明

3. **测试链接**
   - 点击社交账号图标（应该在新标签页打开）
   - 点击产品链接（应该跳转到对应页面）
   - 点击开发者资源链接（应该跳转到开发者页）
   - 点击法律信息链接（应该跳转到对应页面）

4. **检查样式**
   - Footer应该是深色背景（bg-gray-900）
   - 链接应该有hover效果（鼠标悬停时变白色）
   - 布局应该是响应式的（移动端和桌面端）
