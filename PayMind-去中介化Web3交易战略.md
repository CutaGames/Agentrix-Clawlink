## 1. 战略定位

### 1.1 核心定位

**PayMind = Web3 原生发行与交易基础设施**

不是要"取代"传统平台，而是提供**基础设施层**，让：
- ✅ **项目方**可以直接发行代币，无需通过 Launchpad
- ✅ **创作者**可以直接发行 NFT，无需通过 NFT 平台
- ✅ **传统平台**也可以接入，获得 AI Agent 流量
- ✅ **用户**通过 Agent 直接与项目方/创作者交易

### 1.2 双重策略

#### 策略1：去中介化（直接发行）

**目标**：让项目方和创作者直接发行和交易

**优势**：
- 降低发行成本
- 提高收益分成
- 更快的发行速度
- 更好的控制权

#### 策略2：平台接入（生态共建）

**目标**：让传统平台也接入，扩大生态

**优势**：
- 快速获得大量商品
- 利用现有用户基础
- 降低获客成本
- 生态网络效应

### 1.3 定位对比

| 维度 | 传统平台 | PayMind |
|------|---------|---------|
| **定位** | 交易平台 | 基础设施层 |
| **发行** | 需要平台审核 | 直接发行 |
| **交易** | 平台内交易 | Agent 驱动交易 |
| **流量** | 平台流量 | AI Agent 流量 |
| **成本** | 平台手续费 | 基础设施费用 |

### 3.3 API 设计

#### 3.3.1 代币发行 API

```typescript
// 创建代币发行请求
POST /api/v1/tokens/launch
Request: {
  name: string              // 代币名称
  symbol: string            // 代币符号
  totalSupply: string       // 总供应量
  decimals: number          // 精度
  chain: 'ethereum' | 'solana' | 'bsc' | 'polygon'
  distribution?: {          // 分配方案
    team: number            // 团队比例
    investors: number       // 投资者比例
    public: number          // 公开销售比例
    reserve: number         // 储备比例
  }
  lockup?: {                // 锁仓机制
    team: {                  // 团队锁仓
      amount: number
      releaseSchedule: Array<{
        date: string
        amount: number
      }>
    }
  }
  presale?: {                // 预售设置
    price: number
    startDate: string
    endDate: string
    whitelist?: string[]     // 白名单
  }
}
Response: {
  tokenId: string            // PayMind 代币 ID
  contractAddress: string    // 合约地址
  transactionHash: string    // 部署交易哈希
  productId: string          // 商品 ID（自动创建）
  status: 'deploying' | 'deployed' | 'failed'
}

// 查询代币发行状态
GET /api/v1/tokens/:id/status
Response: {
  status: 'deploying' | 'deployed' | 'failed'
  contractAddress?: string
  transactionHash?: string
  deployedAt?: Date
  error?: string
}
```

### 4.2 智能合约设计

#### 4.2.1 代币直接销售合约

```solidity
// TokenSale.sol
contract TokenSale {
  IERC20 public token;
  IERC20 public paymentToken;  // USDC/USDT
  address public seller;        // 项目方地址
  uint256 public price;         // 代币价格
  uint256 public totalSupply;    // 总供应量
  uint256 public sold;           // 已售数量
  
  function buy(uint256 amount) external {
    require(sold + amount <= totalSupply, "Insufficient supply");
    uint256 payment = amount * price;
    paymentToken.transferFrom(msg.sender, seller, payment);
    token.transfer(msg.sender, amount);
    sold += amount;
  }
  
  function setPrice(uint256 newPrice) external {
    require(msg.sender == seller, "Only seller");
    price = newPrice;
  }
}
```

#### 4.2.2 NFT 直接销售合约

```solidity
// NFTSale.sol
contract NFTSale {
  IERC721 public nft;
  IERC20 public paymentToken;  // USDC/USDT
  address public creator;       // 创作者地址
  uint256 public royalty;       // 版税比例（基点，如 1000 = 10%）
  
  mapping(uint256 => uint256) public prices;  // tokenId => price
  mapping(uint256 => bool) public listed;     // tokenId => isListed
  
  function buy(uint256 tokenId) external {
    require(listed[tokenId], "Not listed");
    uint256 price = prices[tokenId];
    
    // 计算版税
    uint256 royaltyAmount = price * royalty / 10000;
    uint256 sellerAmount = price - royaltyAmount;
    
    // 支付
    paymentToken.transferFrom(msg.sender, creator, royaltyAmount);
    paymentToken.transferFrom(msg.sender, nft.ownerOf(tokenId), sellerAmount);
    
    // 转移 NFT
    nft.transferFrom(nft.ownerOf(tokenId), msg.sender, tokenId);
    
    listed[tokenId] = false;
  }
  
  function list(uint256 tokenId, uint256 price) external {
    require(nft.ownerOf(tokenId) == msg.sender, "Not owner");
    prices[tokenId] = price;
    listed[tokenId] = true;
  }
}
```
