## 2. 统一发布流程（问题2）

### 2.1 流程结构（最终定稿）

**所有专业用户统一走这一套**

- Step 0｜选择你的身份（API / 数据 / 专家 / 全能开发者）
- Step 1｜描述你提供的能力（30–60 秒）
- Step 2｜确认使用方式与价格（60–90 秒）
- Step 3｜发布范围与合规（30 秒）
- 立即发布，可被交易

### 2.2 定价与分佣（硬规则）

**定价**（必须）
- 每个 Skill / 能力必须有价格
- 允许：免费 / 按次 / 订阅
- **系统给推荐价，用户只需确认**

**分佣**（平台统一）
- 用户不设置
- 平台统一：10%
- 页面固定文案：
  - 平台将收取 10% 服务费，用于结算与分发
- 不显示拆解比例

## 3. 四类角色逐字文案（最终版）

### 3.1 API 厂商

**Step 1｜你已经有 API 吗？**
- 标题：让 Agent 直接使用你的 API
- 说明：你不需要编写任何新代码。Agentrix 会自动把你的 API 转换为 Agent 可调用的能力。
- 输入占位：粘贴你的 API 请求地址，或直接粘贴 curl 命令
- 辅助说明：我们只用于结构识别，不会保存你的密钥。
- 主按钮：继续

**Step 2｜系统已理解你的能力**
- 标题：这是 Agent 能理解的能力描述
- 自动生成（可编辑）：
  - 能力名称：查询订单物流状态
  - Agent 可能会在这些情况下使用它：
    - ☑ 帮我查一下这个订单到哪了
    - ☑ 这个包裹什么时候能到？
- 提示：Agentrix 会自动完成对接与分发。
- 按钮：看起来没问题，继续

**Step 3｜价格与发布**
- 标题：确认价格并发布
- 价格卡片：
  - 单次调用价格
  - 系统建议：$0.01 / 次（可修改）
- 免费额度（可选）：前 100 次免费
- 服务费说明（固定文案）：平台将收取 10% 服务费，用于结算与分发
- 按钮：发布并开始被 Agent 使用

---

### 3.2 数据提供方

**Step 1｜你的数据可以帮助 Agent 做什么？**
- 标题：让 Agent 使用你的数据回答问题
- 说明：你不需要暴露原始数据。Agentrix 只会按你的规则返回结果。
- 其余步骤沿用统一流程
