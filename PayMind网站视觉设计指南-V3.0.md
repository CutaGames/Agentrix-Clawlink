# PayMind 网站视觉设计指南 V3.0
## 快速参考指南

**版本**: 3.0  
**日期**: 2025年1月

---

## 🎨 核心设计原则

1. **Agent 优先**：PayMind Agent 作为核心入口
2. **基础设施定位**：强调为 AI Agent 提供完整的交易能力基础设施
3. **AI 驱动**：强调智能推荐和自动执行
4. **用户友好**：对话式交互简化复杂功能
5. **场景化展示**：通过使用场景展示不同用户的价值，包括 Web3 发行与交易能力（但不作为核心定位）

---

## 🧩 组件设计规范

### 按钮

**主要按钮**：
- 背景：渐变（Blue-600 → Indigo-600）
- 文字：白色
- 圆角：8px
- 内边距：16px 32px
- 悬停：阴影增强，轻微放大

**次要按钮**：
- 背景：透明
- 边框：2px 白色
- 文字：白色
- 圆角：8px
- 内边距：16px 32px
- 悬停：背景变白，文字变深色

### 卡片

**标准卡片**：
- 背景：白色
- 阴影：shadow-md
- 圆角：8px
- 内边距：24px
- 悬停：shadow-lg，轻微上移

**功能卡片**：
- 背景：白色
- 阴影：shadow-sm
- 圆角：12px
- 内边距：32px
- 图标：48px，居中
- 标题：H3，居中
- 描述：Body，居中

### 表格

**对比表格**：
- 背景：白色
- 边框：1px Gray-200
- 表头：Gray-100 背景
- 文字：Gray-900
- 圆角：8px

### 导航

**顶部导航**：
- 背景：白色（首页深色背景时透明）
- 高度：64px
- 文字：Gray-900（首页白色）
- 悬停：Indigo-600

**下拉菜单**：
- 背景：白色
- 阴影：shadow-lg
- 圆角：8px
- 内边距：8px

## 📱 响应式断点

- **Mobile**：< 640px
- **Tablet**：640px - 1024px
- **Desktop**：> 1024px

### 移动端适配

- **Hero Section**：单列，标题和按钮垂直排列
- **功能卡片**：单列，每个卡片占满宽度
- **导航**：汉堡菜单
- **表格**：横向滚动
