## 1. 问题分析

### 1.1 核心问题

**问题**：如果通道费用全部由商户承担，向用户显示通道成本是否有意义？

**关键考虑因素**：
1. **用户体验**：用户是否需要看到这些信息？
2. **透明度**：显示成本是否增加信任？
3. **商户灵活性**：商户是否需要将成本转嫁给用户？
4. **商业逻辑**：不同场景是否需要不同策略？

### 1.2 显示通道成本的利弊分析

#### 1.2.1 显示通道成本的优点

**优点**：
1. **透明度**：增加用户信任，显示平台诚实
2. **教育价值**：帮助用户理解不同支付方式的差异
3. **选择依据**：用户可以根据成本选择支付方式（如果商户允许）
4. **差异化展示**：突出低成本支付方式的优势

#### 1.2.2 显示通道成本的缺点

**缺点**：
1. **信息过载**：用户可能不需要这些技术细节
2. **混淆风险**：用户可能误以为需要支付这些费用
3. **决策负担**：增加用户选择负担
4. **界面复杂**：增加界面复杂度

### 1.3 用户心理分析

**用户关注点**：
1. **最终价格**：用户最关心的是最终需要支付多少钱
2. **支付便捷性**：支付方式是否方便
3. **安全性**：支付是否安全
4. **速度**：支付是否快速

**用户不关心**：
1. 通道成本（如果不需要支付）
2. 商户收入
3. 平台分成

### 2.1 方案1：不显示通道成本（简化版）

## 3. 推荐方案

### 3.1 推荐方案：方案2（商户可设置不同支付通道的价格）

#### 3.1.1 推荐理由

**核心优势**：
1. **商户灵活性**：商户可以根据成本设置价格，实现成本转嫁
2. **用户选择权**：用户可以选择更便宜的支付方式
3. **成本透明**：用户知道为什么价格不同，增加信任
4. **差异化竞争**：低成本支付方式可以更低价格，形成竞争优势

**符合行业惯例**：
- **电商平台**：Amazon、eBay等允许商户设置不同支付方式的价格
- **支付平台**：PayPal、Stripe等支持商户设置价格差异
- **用户习惯**：用户已经习惯不同支付方式可能有不同价格

#### 3.1.2 实现策略

**策略1：智能默认价格**
- 默认：所有支付方式使用相同价格（商户承担通道费用）
- 可选：商户可以为高成本通道设置更高价格

**策略2：价格差异提示**
- 显示价格差异原因（通道费用）
- 推荐低成本支付方式
- 说明费用承担方

**策略3：简化显示**
- 主要显示：支付金额
- 次要显示：通道费用（如果用户需要承担）
- 隐藏显示：商户承担的费用（可选）

### 3.2 备选方案：方案1（不显示通道成本）

#### 3.2.1 适用场景

**适用场景**：
1. **标准化商品**：商品价格固定，不需要差异化
2. **用户体验优先**：追求最简洁的用户体验
3. **商户成本可控**：商户可以承担所有通道费用

**实现建议**：
- 默认不显示通道成本
- 商户可以在后台查看成本分析
- 智能路由自动选择最优通道

### 3.3 高级方案：方案3（混合模式）

#### 3.3.1 适用场景

**适用场景**：
1. **高成本通道**：通道费用较高，商户希望分担
2. **灵活定价**：商户需要灵活的定价策略
3. **成本敏感**：商户对成本非常敏感

**实现建议**：
- 默认商户承担
- 商户可以设置用户承担比例
- 清晰显示费用承担方

```typescript
interface PaymentChannelPrice {
  channelId: string;
  basePrice: number;        // 基础价格
  channelFee: number;      // 通道费用
  finalPrice: number;      // 最终价格
  feePaidBy: 'merchant' | 'user' | 'split';  // 费用承担方
  userFeeRatio?: number;   // 用户承担比例（如果split）
}
```

**功能2：价格计算逻辑**
```typescript
function calculateFinalPrice(
  basePrice: number,
  channelFee: number,
  feePaidBy: 'merchant' | 'user' | 'split',
  userFeeRatio?: number
): number {
  if (feePaidBy === 'merchant') {
    return basePrice;  // 用户支付基础价格
  } else if (feePaidBy === 'user') {
    return basePrice + channelFee;  // 用户支付基础价格 + 通道费用
  } else if (feePaidBy === 'split') {
    const userFee = channelFee * (userFeeRatio || 0.5);
    return basePrice + userFee;  // 用户支付基础价格 + 部分通道费用
  }
  return basePrice;
}
```

### 4.3 智能推荐逻辑

#### 4.3.1 推荐算法

**推荐优先级**：
1. **价格最低**：优先推荐用户支付金额最低的方式
2. **速度最快**：如果价格相同，推荐速度最快的方式
3. **成功率最高**：如果价格和速度相同，推荐成功率最高的方式

**推荐理由**：
- 如果价格相同：强调速度、成功率、安全性
- 如果价格不同：强调价格优势，同时说明速度、成功率
