# Agentrix 支付完善计划 (Agentrix Payment Improvement Plan)

**目标**: 将 Agentrix 支付从“技术原型”提升为“高标准支付产品”，满足商户在线上、线下、AI 自动化等全场景的需求。

---

## 1. 场景覆盖与功能增强

### 1.1 线上场景 (Online Checkout) - P0
- **[已完成] 品牌统一化**: 全面使用 **AX ID** 替代旧有的 PM ID/Agentrix ID，增强品牌辨识度。
- **[已完成] 支付成功页升级**: 支付成功后提供包含 AX ID、审计证据哈希 (Audit Proof) 和交易详情的专业收据。
- **[已完成] 托管收银台 (Hosted Checkout) 优化**: 在 Agent 工作台集成收银台配置与实时预览功能。
- **[已完成] 订阅管理 (Subscriptions)**: 在工作台增加订阅管理面板，支持商户管理计划与用户管理扣款。

### 1.2 线下场景 (Offline/POS) - P1
- **[P1] 动态二维码**: 支持商户生成动态收款码，用户扫码后自动填入金额并唤起钱包。
- **[P2] 商家通知 App**: 提供简单的商家端 App 或 Telegram Bot，实现收款即时语音播报。

### 1.3 AI Agent 场景 (Agent-to-Agent) - P0
- **[已完成] 授权策略可视化**: 让用户能清晰看到给 Agent 的授权限额、有效期和用途。
- **[P1] 意图支付 (Intent-to-Pay)**: 优化自然语言触发支付的成功率，减少用户确认步骤。

---

## 2. 基础设施与合规 - P1
- **[P1] 法币入金 (Fiat On-ramp) 深度集成**: 优化 Transak/Stripe 集成，降低非加密用户门槛。
- **[已完成] 商家对账系统**: 提供数据分析面板、收入趋势图表及 CSV 导出功能。
- **[P2] 风险引擎升级**: 集成链上反洗钱 (AML) 扫描，自动拦截高风险地址。

---

## 3. 开发者体验 (DX) - P2
- **[P2] 交互式文档**: 提供类似 Stripe 的 API 文档，支持在文档中直接发起测试请求。
- **[P2] SDK 简化**: 提供一键集成的 React Hook 和 Vue 指令。

---

## 4. P0/P1 级任务执行清单 (当前阶段)

1.  **[已完成]** 首页页头效果优化：修复 AX ID 在暗色背景下的可见性。
2.  **[已完成]** 全局重命名：将 PM ID / Agentrix ID 统一为 **AX ID**。
3.  **[已完成]** 支付成功页升级：在 `SmartCheckout` 组件中增加专业的成功反馈。
4.  **[已完成]** 授权策略面板优化：提升用户对 Agent 授权的信任感。
5.  **[已完成]** 商家对账系统：在工作台集成财务分析与导出功能。
6.  **[已完成]** 订阅管理面板：支持商户与用户的双端订阅管理。
7.  **[已完成]** 收银台配置：支持商户自定义品牌颜色与 Logo，并提供实时预览。
