# 移动端重构方案 B：侧滑空间法 (Spatial Drawer) 详解

> **核心理念**：将日常“消费流量”与硬核“管理流量”在物理空间上彻底折叠。像 Discord 或 Slack 一样，主屏留给沟通和内容，侧滑留给强大的工作台面板。

---

## 1. 空间与导航层级架构

引入 `@react-navigation/drawer` 作为应用的超顶层侧滑抽屉。

```text
[ Root Drawer Navigator ] —— 处理全局左滑/右滑
  │
  ├── Left Drawer (纯抽屉区域) 
  │    └── 🤖 AgentConsoleScreen (原来的后台管控中心，直接铺满抽屉)
  │
  └── Center Screen (主视口区域)
       └── [ Main Tab Navigator ]
             ├── 💬 AgentChat (默认首屏：沉浸式多会话对话界面)
             ├── 🌍 Discover (中间 Tab：技能与动态发现)
             └── 👤 Me (右侧 Tab：个人钱包、收益与系统设置)
```

## 2. 最终 UI Layout 与交互拆解

### 2.1 Center Screen (主视角首屏 - 极简聊天页)

用户一打开 App，看到的是最极简、最直接的 Chat 画面（但左上角多了一个引路人）。

```text
===================================================
[顶部 Header (ChatBar)]
  [ 🍔 (或🤖头像带圈) ]        <- 【新增入口】点击或从屏幕左边缘右滑，拉出宇宙
        Agent Name           <- 居中 
              📞 对讲功能      <- 右侧
---------------------------------------------------
[多会话 Tab 栏 (如果开启多个 Chat)]
===================================================
[ 聊天主要区域 List ] ...
===================================================
[极简输入框 ] [+] [输入...] [🎤]
===================================================
[ 底部 Tab 栏 ]
  [ 💬 对话 (高亮) ]     [ 🌍 发现 ]     [ 👤 我的 ]
===================================================
** 悬浮球 (如果需要全局语音唤醒，依然存在于右下角) **
```

### 2.2 Left Drawer (左侧滑工作台——藏在空间背后的“猛兽”)

当用户在任何界面**【向右滑动屏幕左边缘】**，或是点击**【左上角的 🍔 / 🤖头像】**时，主界面会变暗或向右推开，露出这一整面墙的`控制台`。

```text
===================================================
[侧滑抽屉 - Console 专属空间 (占屏幕宽度 80%)]      
---------------------------------------------------
 (如果有多实例，最左边可以有一条细细的服务器列表，类似 Discord)
 ╭──────────────────────────────────────────────╮
 │ 🔋 能源与状态卡片 (原 Console 头部)              │
 │ 🤖 AgentName •(绿色云端)                       │
 │ [========= Token 消耗度 72% =========]       │
 │ 💾 存储: 8/10GB 赠送                       │
 ╰──────────────────────────────────────────────╯
   
 📌 快速核心看板 (Quick Actions Grid 扁平化成列表)
   ├── 🧠 记忆集管理 (Memory Hub)      <-- 流量超大，前置
   ├── ⚙️ 工作流引擎 (Workflows)       <-- 高频调试，前置
   ├── 🛠️ 能力组件包 (Skill Pack)      <-- 随时开闭
   └── 📋 运行日志 (Logs)              <-- 用于 Debug

 💻 外设与物联矩阵 (Devices)
   ├── 🥽 智能穿戴阵列 (WearableHub) 
   ├── 🖥️ 桌面端反控 (DesktopControl)
   └── 📷 扫码与同域智连 (Local Connect)

 🔐 安全与权限管理
   ├── 💳 自主支付白名单 (开启/关闭)
   ├── 🔍 联网搜索权限
   └── 📧 邮件收发权限

 [ 🚀 + 新建/链接另一个 Agent / BYOC ]   <-- 底部常驻按钮
===================================================
```

## 3. 这种交互模式的【优势】与【挑战】

### 🌟 优势 (为何极客和效率用户会狂喜)
1. **彻底解耦了“用”和“管”**：
   * 主界面的 3 个 Tab (`聊天` + `发现` + `我的`) 是典型的 C 端消费体验，非常干净。
   * 左滑抽屉是典型的 B 端/开发者 Dashboard 体验。
2. **逻辑降权而非层级降权**：
   * 之前塞在 `Me -> AgentManagement` 是隐藏了入口（点击 3 次才能找到）。
   * 侧滑抽屉只要 **1 次滑动**，随取随用。就像用 Slack 时滑出群频道一样自然。
3. **多实例切换的完美温床**：
   * 如果用户以后有 3 种不同的 Agent（一个是写代码的，一个是做客服的），抽屉最左侧天然可以容纳一列类似 Discord 服务器的圆形头像来实现“秒切实例”。

### ⚠️ 风险与挑战
1. **返回手势冲突**：iOS 系统的右滑边缘是“返回上一页”，如果在深层页面（比如看一个商品详情），右滑到底该返回还是弹抽屉？
   * *解决办法*：仅在第一级的一级 Tab 根页面响应侧滑抽屉手势；进入内页后锁定该手势，改为正常的侧滑返回。
2. **需要一定的用户教育**：
   * 不常玩海外社交 App 的小白用户可能不知道“左边还藏着东西”。
   * *解决办法*：首次进入 Chat 时，加一个弹框指向左上角图标，并演示一次动态拉开。

## 4. 实施难度排查

如果采用此方案，代码落地的复杂度在 **中等 (Medium)** 层面：
1. 我们不需要新写业务代码（原本的 27 个 Console 页面组件都已经写好了）。
2. 我们需要引入 `yarn add @react-navigation/drawer` 依赖（如果还没装的话）。
3. 我们需要彻底重写根层的 Navigator 层拓扑 —— 从单纯的 `Tab+Stack` 嵌套，变成 `Drawer -> Tab -> Stack` 结构。
