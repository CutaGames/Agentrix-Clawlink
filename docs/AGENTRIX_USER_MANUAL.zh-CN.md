# Agentrix 当前版本用户手册

更新时间：2026-03-20

适用范围：
- 移动端 ClawLink
- Agentrix Desktop
- 本轮 UX 重构后新增的跨端接力、桌面 Spotlight、离线恢复、桌面剪贴板入口

## 1. 产品入口说明

当前版本的核心交互已经从“先进入管理页，再进入对话”调整为“优先对话，管理能力后置”。

你会看到以下主要入口：
- 移动端聊天页：用于日常文字、语音、附件、位置发送
- 移动端桌面发现条：当检测到在线桌面端时，可把当前会话切到桌面继续
- 移动端桌面剪贴板卡片：当桌面端最近有可用剪贴板内容时，可直接插入输入框或复制到手机剪贴板
- 桌面端悬浮球：快速打开聊天面板
- 桌面端 Spotlight：快速唤起轻量输入框，用于临时提问、处理通知结果、基于选中文本继续问答
- 桌面端聊天面板：完整多会话工作区

## 2. 移动端使用说明

### 2.1 发送消息

在聊天页底部输入框输入内容后：
- 点击发送按钮发送文本
- 点击 `+` 打开附件工具栏
- 可添加拍照、相册、文件、位置

### 2.2 语音输入

聊天页左下角可在键盘和语音模式之间切换。

语音模式支持两种交互：
- 点击式：点一下开始，再点一下发送
- 按住式：按住说话，松开发送

### 2.3 离线模式

当网络不可用时，聊天页顶部会出现离线提示条。

离线状态下：
- 你仍然可以继续输入和点击发送
- 消息会进入本地队列
- 网络恢复后系统会自动按顺序补发

注意：
- 离线时不会立刻获得模型回复
- 恢复联网后会自动继续发送，不需要手动重发

### 2.4 加载更早消息

聊天页现在默认优先显示最近消息，提升打开速度。

当历史较长时：
- 上滑到顶部会出现“加载更早消息”入口
- 点击后按批次补充更早历史

### 2.5 桌面接力

当系统检测到你的桌面端在线时，聊天页会显示桌面发现条。

使用方式：
1. 在手机上继续当前对话到需要切桌面的位置
2. 点击“桌面已发现/继续到桌面”横条
3. 系统会把当前会话上下文发送到桌面
4. 桌面端会自动打开聊天面板并恢复该会话

适合场景：
- 手机上开始提问，桌面上继续长文本或文件操作
- 把移动中的会话转到桌面继续工作

### 2.6 桌面剪贴板入口

当桌面端最近有新的剪贴板内容时，聊天页会显示“桌面剪贴板可用”卡片。

卡片支持两个动作：
- `插入`：把桌面剪贴板内容插入到当前输入框
- `复制`：把桌面剪贴板内容复制到手机本地剪贴板

说明：
- 这是当前版本新增的显式入口
- 它解决了之前“桌面有 clipboard 能力，但手机端没有可见入口”的问题

## 3. 桌面端使用说明

### 3.1 悬浮球

桌面端悬浮球是全局快速入口。

基础操作：
- 单击：打开或关闭聊天面板
- 面板隐藏后再次打开：会恢复之前的工作状态

### 3.2 Spotlight

Spotlight 是轻量快速输入层，适合临时问答和通知回看。

当前版本中：
- 可由系统通知动作打开
- 可带入通知结果文本
- 如果系统中有可获取的选中文本，打开时会尝试读取并作为上下文

适用场景：
- 处理后台任务完成结果
- 对选中的一句话、一段文字快速继续问
- 快速输入短问题而不进入完整聊天面板

### 3.3 系统通知与 Spotlight 联动

当后台任务完成并生成通知时：
- 点击通知动作“View in Spotlight”
- 结果文本会自动带入 Spotlight 输入框
- 你可以直接继续追问、翻译、总结或改写

### 3.4 从移动端接入桌面会话

当移动端发起 handoff 后：
- 桌面端会收到跨设备会话事件
- 自动打开聊天面板
- 自动恢复最近的会话上下文
- 会创建新的桌面会话标签继续处理

### 3.5 剪贴板与选中文本

当前版本优化了两件事：
- 获取选中文本时不再破坏你的原系统剪贴板
- clipboard sync 重复初始化后不再产生重复监听和重复事件

## 4. 跨端协同说明

### 4.1 当前已经可用的能力

- 手机发现桌面在线
- 手机把当前会话切到桌面继续
- 桌面收到会话后自动恢复上下文
- 桌面结果可通过通知带入 Spotlight
- 手机可以看到最近桌面剪贴板内容并显式使用

### 4.2 当前版本的边界

以下内容需要明确：
- 当前跨端主要走统一 presence + handoff 路径
- 局域网直连发现相关底层能力仍在演进中
- 因此“同局域网自动直连”不是当前版本的主交付方式

## 5. 常见问题

### 5.1 为什么我在手机上看不到桌面接力条？

请确认：
- 你的桌面端已经登录同一账号
- 桌面端在线并能正常发送心跳到后端
- 当前网络环境可以访问后端 API

### 5.2 为什么看不到桌面剪贴板卡片？

请确认：
- 桌面端最近确实复制过文本
- 桌面端心跳和上下文同步正常
- 剪贴板内容不是空字符串

### 5.3 离线发送后为什么没立刻有回复？

因为离线状态下消息会先进入本地队列，只有恢复联网后才会真正发送到服务端并获得回复。

## 6. 本次重构后的建议用法

推荐日常路径：
1. 手机端快速开始对话
2. 需要长文本或复杂上下文时，点击桌面接力条
3. 桌面端继续处理，必要时通过 Spotlight 快速二次问答
4. 若桌面复制了可用文本，回到手机端可直接插入当前输入框

这也是当前版本最稳定、最完整的跨端使用路径。
