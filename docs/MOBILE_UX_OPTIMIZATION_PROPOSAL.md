# Agentrix Mobile 各页面终端最终态 Layout & 交互梳理 (v3.0)

> **版本**: v3.0  |  **日期**: 2026-03-27  |  **作者**: 交互设计 & 产品分析  
> **基础**: 基于现有的 2 Tab (Discover/Me) + Agent 全局悬浮球 架构  
> **目标**: 清晰定义各个核心页面的**最终态视觉结构**、**重点样式**与**交互行为**。

---

## 1. 核心导航框架 (全局基座)

App 整体分为**常驻底层导航**与**覆盖全局的交互入口**。

### 1.1 Bottom Tab Bar (底部导航)
*   **结构**: 收敛为 2 个固定入口，高度 64px，顶部 1px border。
*   **[ 🔍 Discover ]**: 左侧 Tab，未选中时 55% 透明度，选中时高亮主题色。
*   **[ 👤 Me ]**: 右侧 Tab，带条件性红点 / 未读数 Badge (位置：右上角绝对定位)。

### 1.2 Global Floating Ball (全局悬浮球)
*   **存在区域**: 除 Chat / VoiceChat / Settings 等沉浸式页面外，全局展示。
*   **默认尺寸与位置**:
    *   **Size**: 48x48 px，完美圆形。
    *   **Position**: 屏幕靠右侧边缘 (Edge margin: 12px)，距离底部 200px 左右。
*   **视觉呈现**:
    *   深夜蓝背景 (`#1a1a2e`)，内部白色字号 16px 加粗显示 `AX` (品牌缩写)。
    *   主外发光 Shadow: `#6C5CE7` 颜色，3px offset, 0.35 opacity, 10px blur。
    *   右下角绝对定位渲染一个小圆点(18x18px) `State Indicator Dot`，用于展示当前 Agent 状态（录音 🎙，思考 `...`，播放 🔊）。
*   **交互定义 (Touch Handlers)**:
    1.  **单击 (Tap)**:
        *   **逻辑**：呼起原生 `navigation.navigate('AgentChat')`，携带参数 `{voiceMode: true, duplexMode: true}`。**不再使用 `CommonActions.reset`**，而是保持原本浏览栈，对话结束后点左上角返回能原路归位（如 Discover 浏览一半）。
    2.  **长按 (Long Press, 400ms)**:
        *   **逻辑**：在上方（ball.top - 10px）弹出一个 `260x120px` 的 **Voice Pill Panel (语音胶囊面板)** 供轻量互动，或直接转入对讲。
    3.  **拖拽 (Pan/Drag)**:
        *   **逻辑**：手指松开后，通过 `Spring Animation` (friction 7) 吸附回屏幕左侧或右侧的最近安全边缘，并将新坐标通过 `MMKV` 持久化记录（下次冷启动在原位）。

*(选做改进：新增用户首次进入时，悬浮球旁飘出 3 秒消失的 Tooltip：“👋 点我开始 AI 对话 | 长按进入实时语音”)*

## 2. Agent Chat Screen (对话打磨核心页)

**设计目标**：砍掉多余弹窗与认知过载元素，追求接近极致的即时通讯 + AI 工具流体验。

### 2.1 整体 Layout (自上而下)

```text
===================================================
[顶部状态栏]
---------------------------------------------------
[ChatBar (NavigationBar)]                        <- 高度 80px (安全区+60)
  < 返回                                           <- 左1: [36x36 圆形背景 Icon]
            Agent 名字 (16px 加粗 居中)            <- 中间: 占据剩余 flex 空间 
  📞 对讲                     ⚙️ 设置              <- 右2: [对讲开关，36x36] / 右1: [设置抽屉入口]
---------------------------------------------------
[多会话 Tab 栏 (ChatSessionTabs)]                 <- 水平横向滚动
===================================================
[消息列表 FlatList]                              <- P1: 背景色 bgPrimary, 内边距 16px
  
   [ 系统 Banner 区: 桌面连接 / 离线提醒等 (条件渲染) ]

   (对方 - Agent)                                 (我方 - User)
   ╭────────────────────────────╮
   │🤖 消息气泡 (bgCard 带边框)  │             ╭────────────────────╮
   │左对齐, 左上右下圆角 (18px)    │             │主色气泡 (primary)  │
   │                            │             │右对齐, 左下左上圆角 │
   ╰────────────────────────────╯             ╰────────────────────╯
   
   [ ⚡ 思维链胶囊 ThoughtChain Capsule ]          <- 默认折叠，背景浅色
   格式: "⚡ 5 steps completed" 或 (在生成中时显示最后一条 thought 的跑马灯)
   点击展开 -> [ ⚡ 调用搜索工具... ] -> [ ❯ 解析网页内容... ]
===================================================
[底部语音状态吸附区 (仅 VoiceMode 开启时渲染)]
  [... 动画波纹] 听写状态态 / 转换中文字预览 (12px灰字, 限2行)
===================================================
[多模态极简输入区 (InputRow)]                     <- 🌟 痛点重构区域
  [ + 附件(42x42)] [      输入框(动态增高)     ] [ 🎤(空时不占版面)/⬆发送(有字时) ]

[系统软键盘区 (iOS 88px offset)]
```

### 2.2 瓶颈优化细节与交互

*   **底部输入区 (InputRow) 动态删减收敛到 3 元素** (告别 5 控件过载)：
    *   **默认无文本态**：`[ ➕ 附件号 ]` `[ 灰底输入框 ]` `[ 🎤 语音入口按钮 ]`
    *   **打字输入态**：`[ ➕ 附件号 ]` `[ 输入框 (随字数长高最多120px) ]` `[ ⬆ 主色发送按钮 ]`
    *   *(取消右侧的 [Copy] 按钮留白给发送，长按输入框自带气泡系统复制；取消左侧专门切模式的切换按钮，空框点击麦克风即切为语音)*。
    *   在使用 语音模式 (VoiceMode=ON + PTT) 时，输入框区域变为**绿色高亮的 `🎙 按住说话`**。

*   **模型切换降维 (告别全屏 Modal)**：
    *   **现状**：点 ⚙️出 Sheet → 点模型 → 关闭 Sheet → 等 300 毫秒动画 → 弹全屏 Modal 选模型。
    *   **最终态交互**：点 ⚙️ 呼出 **Settings Bottom Sheet**。Sheet 中直接罗列 `[语音人格 Zhiyu ▾]`，下方紧接 `[智能模型 GPT-4o ▾]`。点击模型行时，**不弹 Modal**，而在 Sheet 内部直接纵向手风琴展开 (Inline Expandable) 列表项。点击选中后直接收起列表并更新。

*   **ThoughtChain 流式交互**：
    *   在推流阶段，不再干等，胶囊内文本跟随 `thoughts[]` 最后一条实时 `replace` 更新。例如 `[⏳ 正在搜索...]` -> `[⏳ 提取页面...]` -> 结束变为固定的 `[ ✓ 5 steps completed ]`。

```text
  (Skills 选中底色亮起)      (Showcase 灰态)

  [ 技能双格瀑布流 (47% width / gap 12px) ]
```

### 3.2 交互修改
*   **Category Chips 的本地筛选 (不再生硬跳转)**:
    *   现状：点 `⚡热门` Chip，整个页面切栈跳转到了 Marketplace 去查询。
    *   改进：变成同页过滤器状态机 (`activeFilter`)。选择不同 Chip 只改变其视觉高亮并触发底下的**技能双格瀑布流**列表内容进行本地替换/加载，保持页面稳定性。

## 5. Launch/Onboarding (用户新入初始态)

### 5.1 Auth 与 部署引导流
1.  **Auth 层**：未登录打回 Login + WalletConnect。成功登陆后检查 `hasCompletedOnboarding`。
2.  **Deploy Select 页面 (部署网关)**：新用户强制出现。
    *   白底视图 `Padding 24px`，大标题 `Agentrix Claw (28px 800)`。
    *   3 个纵向卡片供选排列 (带选中边框高亮)：
        *   ☁️ **One-Tap Cloud Deploy** (绿色角标 10GB FREE)
        *   💻 **Local / Private Deploy** (主色角标 PRIVATE)
        *   ⚙️ **BYOC** (带置灰说明)
    *   底部拥有次级按钮：`[ Skip for now ]`。

### 5.2 返回栈架构层 (`AgentStackNavigator`)
```javascript
// 该逻辑支撑 方案1：首屏变更
// 现状：
initialRouteName: 'AgentConsole' // 新手进来会进满是数据的运维面板
// 改为：
initialRouteName: 'AgentChat'    // 新手一旦配置或Skip，主进入即为 ChatUI
// 在 AgentChat 中：如果是新用户还没 Agent，利用 S2 已写好的判断逻辑，
// 会在这个无 Header 界面展示 [大外星人头像 + Get Started] 引发自动部署 API (`POST /openclaw/auto-provision`)。
```
