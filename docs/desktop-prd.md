# Agentrix Desktop — 产品需求文档 (PRD)

**版本**: v1.0  
**日期**: 2026-03-08  
**产品名**: Agentrix Desktop (代号: ClawBall)  
**定位**: 桌面端 AI Agent 入口 — 悬浮球 + 语音唤醒 + 跨设备协同

---

## 一、产品愿景

> **让用户在任何时间、任何地点、跨任何设备，与 Agent 无缝交互。**

Agentrix 的多端终极交互形态：

| 端 | 形态 | 核心场景 |
|---|------|---------|
| **移动端** | 全功能 App (ClawLink) | 随身携带，碎片时间，外出场景 |
| **桌面端** | 悬浮球 + 语音面板 (ClawBall) | 工作场景，深度任务，企业协作 |
| **Web 端** | 管理控台 | 配置管理，数据分析，商户后台 |

桌面端不是"又一个聊天窗口"，而是**操作系统级别的 AI 入口**——像手机里的 Siri/Assistant 一样，桌面上的一个悬浮小球，语音唤醒，随时对话，随时交代任务。

---

## 二、用户画像

### 个人用户 (Consumer)
- **核心诉求**: 随时随地与 AI Agent 对话，交代任务，无需打开任何应用
- **使用场景**:
  - 工作时一句话让 Agent 搜索资料、整理文档
  - 语音交代"帮我查一下今天的航班"
  - 边看网页边让 Agent 总结
  - 手机上开始的任务，回到电脑继续
- **期望体验**: 像跟助手说话一样自然，零学习成本

### 企业/专业用户 (Enterprise)
- **核心诉求**: 专业级 Agent 工作台，团队协作，数据安全
- **使用场景**:
  - 多 Agent 编排工作流 (研究Agent → 写作Agent → 审核Agent)
  - 私有数据 RAG 问答 (本地文档不出域)
  - 团队共享 Agent 配置和技能
  - 技能开发 IDE + 调试 + 发布
- **期望体验**: 专业、高效、安全、可控

### 技术选型: Tauri 2.0

| 为什么选 Tauri | 说明 |
|---------------|------|
| **体积小** | 安装包 8-15 MB (vs Electron 150+ MB) |
| **性能好** | Rust 后端，内存占用 30-50 MB (vs Electron 200+ MB) |
| **系统集成** | 原生全局热键、托盘、窗口管理 |
| **与现有协同** | 前端复用 React/TS，后端复用同一套 API |
| **安全** | Rust 内存安全，CSP 策略，IPC 权限系统 |
| **自动更新** | 内置 Updater 插件 |

## 四、功能规格

### P0 — MVP (首版发布, 4 周)

> **核心目标**: 悬浮球 + 对话 + 语音，可独立使用

#### 4.1 悬浮球 (Floating Ball)

| 项目 | 规格 |
|------|------|
| **形态** | 56×56px 圆形半透明球体，Agentrix logo |
| **位置** | 默认右下角，可拖拽到屏幕任意边缘，位置跨重启记忆 |
| **层级** | Always-on-top，悬浮于所有窗口之上 |
| **状态指示** | 默认态(静止) / 聆听态(呼吸动画) / 思考态(旋转) / 说话态(波纹) |
| **交互** | 单击→展开面板 / 长按→语音输入 / 右键→快捷菜单 / 拖拽→移动位置 |
| **自动吸附** | 松手后吸附到最近屏幕边缘，半隐藏（露出半球） |
| **透明度** | 默认 80% 不透明，鼠标悬停 100%，空闲 30 秒渐变为 40% |

#### 4.2 对话面板 (Chat Panel)

| 项目 | 规格 |
|------|------|
| **展开方式** | 从悬浮球位置弹出，480×640px 面板（可拖拽调整大小） |
| **消息类型** | 文本 / Markdown / 代码块 / 思维链 / 错误状态 |
| **输入方式** | 文本输入框 + 语音按钮 + 文件拖入 |
| **流式输出** | SSE 逐字显示，与移动端相同的流式体验 |
| **会话管理** | 新建会话 / 历史会话列表 / 跨设备同步 |
| **模型切换** | 顶部下拉选择 AI 模型 (Claude/GPT/Gemini/Groq/DeepSeek) |
| **快捷键** | Enter 发送, Shift+Enter 换行, Esc 收起面板, Ctrl+N 新会话 |
| **收起** | 点击面板外区域 或 Esc 键，收回悬浮球 |

#### 4.3 语音交互 (Voice)

| 项目 | 规格 |
|------|------|
| **唤醒方式** | 全局热键 (默认 Ctrl+Shift+A) / 长按悬浮球 / 面板内语音按钮 |
| **录音** | 按住说话 (PTT) 模式，松开自动发送 |
| **STT** | 发送音频到 `/voice/stt` 转文字，显示在输入框 |
| **TTS** | Agent 回复自动语音播放，可在设置中关闭 |
| **状态反馈** | 悬浮球呼吸动画(录音) → 旋转(思考) → 波纹(播放语音) |
| **取消** | 录音中按 Esc 取消 |

#### 4.4 认证 & 实例连接

| 项目 | 规格 |
|------|------|
| **登录方式** | 邮箱验证码 / 移动端扫码授权 / OAuth (Google/Twitter/Discord) |
| **Token 存储** | OS 密钥链 (Windows Credential Manager / macOS Keychain) |
| **实例自动发现** | 登录后自动拉取已绑定的 OpenClaw 实例列表 |
| **本地 Agent** | 检测本地 7474 端口是否有 OpenClaw 运行，自动连接 |
| **首次引导** | 无实例时显示: ① 一键部署云端 ② 安装本地 Agent ③ 手动输入地址 |

#### 4.5 系统集成

| 项目 | 规格 |
|------|------|
| **系统托盘** | 右键菜单: 显示/隐藏悬浮球, 设置, 退出 |
| **开机自启** | 可选，默认开启 |
| **全局热键** | Ctrl+Shift+A 激活语音, Ctrl+Shift+S 展开面板 |
| **自动更新** | Tauri Updater，静默检查 + 用户确认安装 |
| **多显示器** | 悬浮球跟随活动显示器 |
| **平台支持** | Windows 10+ / macOS 12+ / Linux (Ubuntu 22+) |

### P1 — 体验增强 (第 2 个月)

#### 4.6 跨设备会话同步
- 手机发起的对话，桌面端实时同步显示
- 桌面端发起的任务，手机端通知进度
- 共享 sessionId，通过后端 `/openclaw/proxy/{id}/history` 同步

#### 4.7 快捷命令 (Slash Commands)
```
/search <query>       — Agent 搜索
/skill <name>         — 执行指定技能
/model <name>         — 切换模型
/new                  — 新建会话
/history              — 查看历史
/clear                — 清空当前会话
```

#### 4.8 剪贴板智能
- 复制文本后，悬浮球出现小徽章提示"已捕获"
- 点击徽章: 选择 "翻译 / 总结 / 解释 / 改写"

#### 4.9 文件拖拽
- 拖拽文件到悬浮球/面板，自动上传并分析
- 支持: PDF, Word, Excel, 图片, 代码文件
- 图片→OCR→分析; 文档→提取文本→总结

#### 4.10 通知中心
- 系统通知: Agent 任务完成、技能市场更新、收入通知
- 与移动端推送共享通知源 (`/notifications/recent`)
- 悬浮球右上角红点提示

### P2 — 企业专业版 (第 3-4 个月)

#### 4.11 多 Agent 编排
- 拖拽式工作流设计器 (DAG 图)
- Agent 节点: 输入→处理→输出，连线定义数据流
- 模板库: 研究→写作→审核流水线, 客服自动分流 等
- 实时执行监控 + 日志面板

#### 4.12 技能开发 IDE
- 内置代码编辑器 (Monaco Editor)
- 技能模板脚手架一键生成
- 本地调试沙箱 (连接本地 OpenClaw)
- 一键发布到 Marketplace

#### 4.13 私有部署模式
- 本地 LLM 接入 (Ollama / llama.cpp)
- 数据不出域，所有对话本地存储
- 企业 SSO (SAML / OIDC)
- 管理员控制台: 用户管理 / 权限 / 审计日志

#### 4.14 团队协作
- 共享 Agent 配置 (团队空间)
- 对话共享 & 协同标注
- @提及队友 + Agent
- 角色权限: Admin / Developer / Viewer

#### 4.15 数据分析面板
- Agent 使用统计 (调用次数/Token消耗/成本)
- 技能 ROI 分析
- 团队效率报告
- 自定义仪表盘

## 六、API 集成规格

桌面端**完全复用现有后端 API**，无需新增后端接口。

### 6.1 认证
```
POST /api/auth/email/send-code      — 发送验证码
POST /api/auth/email/verify-code    — 验证登录
GET  /api/auth/me                   — 获取用户信息 + 实例列表
Headers: { Authorization: "Bearer {token}" }
Token 存储: OS Keychain (Tauri plugin-keychain)
```

### 6.2 Agent 通信
```
POST /api/openclaw/proxy/{instanceId}/stream   — SSE 流式对话 (主要)
POST /api/claude/chat                          — 直接对话 (降级)
GET  /api/openclaw/proxy/{instanceId}/history   — 会话历史
GET  /api/openclaw/instances                    — 实例列表
PATCH /api/openclaw/instances/{id}/model        — 切换模型
```

### 6.3 语音
```
POST /api/voice/stt     — 语音转文字 (上传音频)
GET  /api/voice/tts     — 文字转语音 (返回音频流)
```

### 6.4 通知
```
GET /api/notifications/recent?limit=10&since={ts}  — 轮询通知
```
