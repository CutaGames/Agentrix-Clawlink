## 3. 运营看板设计

### 报告模板

| 频率 | 内容 |
|------|------|
| 日报 | 昨日进展、今日计划、阻碍事项、关键数据（WAA/注册/活跃/成本）|
| 周报 | 本周目标完成度、关键数据趋势、问题与改进、下周计划 |
| 月报 | OKR 进展、核心指标、预算执行、风险与机会、下月重点 |

### 核心数据指标定义

| 指标 | 定义 |
|------|------|
| WAA | Weekly Active Agent — 周活跃代理用户数 |
| DAU | Daily Active User — 日活跃用户数 |
| MRR | Monthly Recurring Revenue — 月度经常性收入 |
| CAC | Customer Acquisition Cost — 获客成本 |
| LTV | Life Time Value — 用户生命周期价值 |

### 数据来源

- 平台数据库（用户、收入、API 调用）
- 社区平台（Discord/Telegram/GitHub）
- 第三方服务（AWS、Solana、API 账单）
- 各 Agent 周报/日报

## 4. 成本追踪

### 当前成本基线

| 项目 | 金额 | 备注 |
|------|------|------|
| AWS Credits | $2,000 | 有效期至 2026-09 |
| API 成本 | ~$100/月 | 按月结算 |
| 服务器 | $50/月 | 新加坡主节点 |

### Q2 预计支出预算

| 类别 | 预算 |
|------|------|
| 云服务（AWS/服务器）| $500 |
| API/第三方服务 | $300 |
| 社区/市场推广 | $200 |
| 运营/工具 | $100 |
| 预留应急 | $200 |
| **合计** | **$1,300** |

### 成本优化建议

- 优先用完 AWS credits，减少自付
- API 调用分级限流，监控异常消耗
- 服务器按需扩缩容
- 社区/市场推广优先用免费渠道
