## 4. GitHub 社区管理

### Issue triage 流程

1. 新 Issue 自动分配标签（bug/feature/question）
2. 24h 内社区经理初步回复
3. 复杂问题 @相关模块负责人
4. 每周整理未解决 Issue，定期归档

### PR review 社区贡献者指南

- 代码需通过 CI
- 必须附带描述/截图
- 推荐附测试用例
- 2 人以上 review 通过方可合并

### Good First Issue 策略

- 标记 "good first issue" 标签 + 详细操作指引 + 指定社区导师答疑

### CONTRIBUTING.md 大纲

- 项目介绍 → 环境搭建 → 提交规范 → Issue/PR 流程 → 行为准则 → FAQ

## 5. 社区健康指标

### 社区分级

| 级别 | 名称 | 条件 |
|------|------|------|
| L1 | Newcomer（新手）| 加入社区 |
| L2 | Contributor（贡献者）| 首次有效贡献（PR/Issue/内容）|
| L3 | Champion（活跃骨干）| 持续贡献 3+ 周 |
| L4 | Ambassador（社区大使）| 邀请 10+ 新成员 + 活跃贡献 |

### 管理与反垃圾策略

- Bot 自动检测广告/刷屏
- 举报机制（3 次警告自动禁言）
- 定期清理不活跃账号
- "社区守护者"志愿者机制
