## 1. 起始资金建议

### 1.1 建议注入金额

- **推荐金额：$300 USDC**
- **理由：**
  - $100 过低，难以分散多渠道、覆盖 gas 及 LP 最低门槛
  - $500 虽更灵活，但初期风险控制优先，$300 足以覆盖多策略试点
  - 便于日亏损/周亏损风控阈值管理

### 1.2 推荐入金方式

- **链：Solana**
- **理由：**
  - 交易费用极低（< $0.01/笔），适合高频操作
  - DeFi 生态成熟，主流协议丰富（Marinade, Jito, Raydium, Orca 等）
  - 钱包与 USDC 流动性充足，安全性高
- **备选链：Base**
  - 费用低，生态新兴，适合后续扩展
- **入金方式：**
  - CEX（如 Binance）USDC 提现至 Solana 钱包
  - 或通过官方桥接（如 Wormhole）跨链转入

## 3. 收入渠道详细计划

### 3.1 Staking & Lending（低风险）

| 协议 | 链 | 预期年化 | 说明 |
|------|----|---------|----- |
| Marinade | Solana | 6-8% | SOL Liquid Staking |
| Jito | Solana | 7-9% | SOL MEV-boosted Staking |
| Solend | Solana | 3-6% | USDC Lending |
| Kamino | Solana | 4-7% | USDC Lending + Yield |
| Aave | Base | 3-5% | USDC Lending（后续扩展） |

**操作步骤：**
1. 将 ~$60 USDC 换为 SOL
2. 通过 Marinade/Jito 质押 SOL，获得 mSOL/jitoSOL
3. 剩余 $120 USDC 存入 Solend/Kamino 获得利息
4. 定期复查收益，动态调整仓位

### 3.2 Bounties & Tasks（低风险）

| 平台 | 类型 | 日均收益预期 | 说明 |
|------|------|------------|------|
| Gitcoin | 开发/设计赏金 | $1-5 | 较高质量任务 |
| Layer3 | 链上任务 | $0.5-2 | 简单交互任务多 |
| Galxe | 社交/链上任务 | $0.5-1 | 数量多、门槛低 |
| Zealy | 社区任务 | $0.5-1 | 签到/社交/推广 |
| QuestN | 链上任务 | $0.5-1 | 新兴平台 |

**选择标准：**
- 任务门槛低、周期短、奖励稳定
- 优先选择新手任务、签到任务
- 关注高潜力新项目的早期任务

### 3.3 Airdrop Farming（中风险）

| 协议 | 链 | 交互成本 | 预期回报 | 确定性 |
|------|------|---------|---------|--------|
| Jupiter | Solana | $1-3 | $20-200 | 中 |
| Kamino | Solana | $1-3 | $10-100 | 中 |
| Drift | Solana | $2-5 | $20-200 | 中 |
| MarginFi | Solana | $1-3 | $10-150 | 中 |
| Base 新协议 | Base | $1-5 | $10-100 | 低-中 |

**风险控制：**
- 单项目投入不超过 $15
- 仅使用白名单主流协议
- 谨防钓鱼合约

### 3.4 DEX LP（中风险）

| LP对 | DEX | APR预期 | IL风险 |
|------|-----|---------|--------|
| USDC/USDT | Raydium | 5-15% | 极低 |
| USDC/USDT | Orca | 4-12% | 极低 |
| SOL/USDC | Raydium | 15-30% | 中 |

**IL 风险控制：**
- 初期仅做稳定币对（USDC/USDT）
- 定期监控 LP 收益与本金
- SOL/USDC 对需 CEO 审批后开放

### 3.5 预测市场（高风险，CEO 批准后）

- **平台：** Polymarket
- **策略：** 仅参与高确定性（>80% 概率）事件
- **仓位限制：** 单事件 ≤ $10，总投入 ≤ $30
- **止损：** 单事件亏损 > 50% 立刻退出

## 4. 日/周运营流程

### 4.1 每日检查清单

- [ ] 检查钱包余额与各协议仓位
- [ ] 统计昨日收益/亏损，核对风控阈值
- [ ] 监控 staking/lending 收益率变化
- [ ] 浏览 bounty/task 平台新任务
- [ ] 检查 airdrop 项目交互进度
- [ ] 记录所有操作与收益（PnL 日志）

### 4.2 每周复盘模板

```markdown
## Treasury 周报 [W__]
- 期初余额: $___
- 期末余额: $___
- 本周收益: $___ (___%)
- 最大回撤: ___%
- 各渠道收益:
  - Staking/Lending: $___
  - Bounties: $___
  - LP: $___
  - Airdrop: $___
  - Other: $___
- 风险事件: [无/有: 描述]
- 下周调整: [具体计划]
```

### 4.3 异常处理流程

| 触发条件 | 动作 |
|---------|------|
| 日亏损 > 10% | 立即暂停所有交易，汇报 CEO |
| 周亏损 > 20% | 冻结钱包，提交详细复盘 |
| 异常合约/资金流动 | 暂停相关操作，核查安全 |
| 协议遭遇攻击/漏洞 | 立刻撤出资金，冻结该协议交互 |
