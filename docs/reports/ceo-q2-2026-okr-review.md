## 五、风险评估与缓解（CEO 视角）

### Top 3 战略风险

#### 风险 1: Show HN 哑弹 — 用户来了发现"没东西可玩" 🔴

**概率**: 中高（如果 4/7 前 Skills 体验不达标）  
**影响**: 致命 — Show HN 只有一次首发机会，烂了就没有第二次  
**缓解**:
- 4/5 前完成内部全链路测试（注册→创建 Agent→调用 Skill→看到结果）
- 准备 "Plan B" 内容：即使产品不够完善，至少让 Dashboard 展示 11 个 Agent 的实时运营状态（这本身就是最大卖点）
- @qa-ops 在 4/6 做一次完整 Smoke Test，不通过则推迟发布

#### 风险 2: 增长昙花一现 — 首周热度后迅速归零 🟡

**概率**: 高（Show HN/PH 流量天然是脉冲式的）  
**影响**: 高 — 没有留存就没有 WAA，OKR 全部失败  
**缓解**:
- Show HN 当天必须收集邮箱（Waitlist 或 Newsletter）
- Day 1 自动邮件：欢迎 + 快速引导创建首个 Agent
- Day 7 邮件：展示你的 Agent 这周做了什么（需要有留存 Hook）
- 核心留存机制：Agent Bounty Board（每日任务 → Agent 完成 → 赚积分）

#### 风险 3: 资金耗尽前未达 PMF 🟡

**概率**: 中（AWS credits 9 月到期，现金预算有限）  
**影响**: 致命  
**缓解**:
- 严控月支出 ≤ $500（含 Treasury DeFi 资金），6 个月跑道
- YC/Techstars 申请是融资的第一棒
- 5 月启动 Pro 订阅测试，即使只有 10 个付费用户也能验证付费意愿
- AWS Credits 用完前必须拿到替代资源（Azure/GCP Startup Program）
- 最坏情况 Plan C：如果 6 月底 WAA < 50，考虑 Pivot 或寻求天使轮
