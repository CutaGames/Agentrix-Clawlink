# PRD: 移动端本地轻量唤醒词模型

## 背景

当前移动端唤醒词链路分为两类：

1. Picovoice Porcupine 离线热词
2. 系统语音识别 fallback

问题在于：

- Picovoice 免费设备额度不足以覆盖种子用户试用
- 系统语音识别对 `Agentrix` 这类中英混合热词稳定性不够
- 现有产品已经形成两种主交互模式：`按住说话` 与 `实时通话`
- 热词唤醒需要成为前台增强能力，而不是继续依赖第三方配额

## 目标

实现一个完全本地、端侧运行的轻量唤醒词模型，满足：

- 不依赖第三方按设备配额
- 仅在 App 前台运行
- 接入当前悬浮球语音入口
- 允许用户在设置页录制和重置自己的唤醒词样本
- 默认优先级可与 Picovoice / 系统 fallback 共存

## 非目标

- 不做 iOS/Android 后台常驻热词
- 不做多热词平台
- 不做说话人无关的大规模泛化训练
- 不做云端热词识别主链

## 产品策略

短期保留两条链路：

1. `本地轻量模型`：当前版本主推的前台热词链路
2. `系统语音 fallback`：兜底

默认策略：

- `auto` 模式下：`本地模型 > 系统 fallback`
- 用户可在设置页显式切换引擎偏好

## 核心方案

### 1. 模型形态

采用“说话人相关”的端侧模板模型，而非大模型推理。

具体实现：

- 使用现有 `@picovoice/react-native-voice-processor` 获取 `16k PCM`
- 用户录制 1~5 条 `Hey Agentrix` / `Agentrix` 样本
- 将每条样本转换为固定长度的声学指纹向量
- 运行时对每个疑似语音片段计算同类向量，与模板做相似度匹配

### 2. 特征设计

每段音频抽取轻量特征：

- 分段 RMS 能量
- 过零率
- 波形变化率
- 低 / 中 / 高频代表频点能量

最终形成归一化特征向量，用余弦相似度完成匹配。

### 3. 运行机制

- 前台常驻监听，仅在悬浮球可见时启用
- 基于音量阈值检测 speech start / speech end
- 每个完整 utterance 在端侧完成模板匹配
- 命中阈值后直接触发现有 `activateVoiceExperience()`

### 4. 设置页能力

- 选择唤醒词引擎：`auto / local / system`
- 录制本地样本
- 查看样本数
- 自检一次当前本地模型匹配分数
- 清空本地模型

## 平台常驻策略

### 桌面端

- 可实现“关闭主面板后仍保留悬浮球”
- 可通过 tray + always-on-top floating ball 常驻
- 本地唤醒词应运行在桌面常驻进程中，而不是依赖可被销毁的聊天面板

### Android

- 目标是“退到后台或退出主界面后，仍保留系统悬浮球，并继续本地热词监听”
- 这需要原生层能力，而不是 RN 页面逻辑：
  - `SYSTEM_ALERT_WINDOW` 悬浮窗
  - `ForegroundService + microphone` 常驻录音
  - 原生热词检测服务
  - 点击悬浮球后用 deep link / intent 拉起实时语音界面
- 当前仓库尚未具备完整原生热词服务，因此本期只能先完成前台本地唤醒词；后台常驻要进入 Android 原生专项

### iOS

- iOS 不支持 Android 那种系统级悬浮窗常驻
- 即使申请后台音频，也不能按 Android 的交互形态长期保留一个系统悬浮球
- iOS 只能保留前台内悬浮球体验，后台常驻热词不作为当前目标

## 用户流程

### 初次录制

1. 用户进入设置页
2. 选择 `本地模型`
3. 点击 `录制样本`
4. 对着手机说出 `Hey Agentrix`
5. 至少录制 1 条，推荐 3 条
6. 返回首页，悬浮球开始使用本地模型监听

### 命中热词

1. 用户在前台说出唤醒词
2. 本地模型完成模板匹配
3. 命中后悬浮球直接进入当前实时语音链路

## 风险

### 风险 1：不同环境下误唤醒

缓解：

- 设置最小 / 最大 utterance 长度
- 增加触发 cooldown
- 提供 sensitivity 调整

### 风险 2：不同用户之间泛化差

缓解：

- 明确定位为“本机、本用户训练型热词”
- 允许多次录制样本求平均模板

### 风险 3：训练时与悬浮球监听抢占麦克风

缓解：

- 设置页训练时隐藏 / 停用悬浮球热词监听
