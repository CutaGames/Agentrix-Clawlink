# Agentrix 当前版本测试工程师检查清单

更新时间：2026-03-20

目标：
- 覆盖本轮 UX 重构后的关键路径
- 在发布前发现跨端接力、桌面通知、Spotlight、离线、剪贴板入口等回归
- 给测试执行人员一份可直接照着跑的步骤清单

## 1. 测试范围

本清单覆盖：
- 移动端聊天主流程
- 移动端离线与历史加载
- 移动端桌面发现与 handoff
- 移动端桌面剪贴板显式入口
- 桌面端悬浮球、聊天面板、Spotlight
- 桌面端通知联动
- 桌面端 selected text / clipboard 行为
- 后端 `desktop-sync` / `agent-presence` 相关接口回归

## 2. 测试前准备

### 2.1 账号与环境

准备 1 个测试账号，确保：
- 可登录移动端
- 可登录桌面端
- 有可用 Agent 或可直接进入聊天页

### 2.2 设备准备

至少准备：
- 1 台 Android 真机或模拟器
- 1 台桌面端 Windows 测试机

建议附加：
- 第二台移动设备用于复测账户并发状态
- 真实网络波动环境用于离线恢复验证

### 2.3 服务准备

确认：
- 后端已部署最新代码
- 数据库迁移已执行
- 桌面端和移动端都指向同一后端环境

## 3. 自动化回归步骤

### 3.1 API 回归

执行：

```bash
npx playwright test tests/e2e/desktop-sync-approval-agent.spec.ts tests/e2e/cross-platform-regression.spec.ts -c tests/e2e/playwright.config.ts --project=chromium
```

通过标准：
- 除环境型已知问题外，不应出现新的 `500`
- `desktop-sync heartbeat` 必须返回 `< 400`
- `desktop-sync state`、`agent-presence handoffs`、`unified devices` 路径必须可用

### 3.2 桌面 UI smoke

执行桌面 Playwright：

```bash
cd desktop
npx playwright test tests/e2e/app.spec.ts
```

通过标准：
- 主窗口可启动
- 登录/guest/设置/新建对话等基础入口可访问

### 3.3 桌面构建验证

执行：

```bash
cd desktop
npm run build
cd src-tauri
cargo check
```

通过标准：
- 前端 build 通过
- Tauri Rust 编译通过

## 4. 移动端手工测试步骤

### M-01 聊天首路径

步骤：
1. 打开移动端并登录
2. 进入 Agent 聊天页
3. 输入一条普通文本并发送

预期：
- 页面直接可聊天
- 消息正常进入列表
- 能收到模型回复

### M-02 附件工具栏

步骤：
1. 点击 `+`
2. 依次验证拍照、相册、文件、位置入口存在

预期：
- 工具栏可以展开和收起
- 每个入口都能正常触发对应流程

### M-03 语音模式切换

步骤：
1. 点击左下角键盘/麦克风切换
2. 测试点击式和按住式语音

预期：
- 两种模式可切换
- 录音、转写、发送流程不崩溃

### M-04 历史懒加载

步骤：
1. 准备一个超过 30 条消息的会话
2. 重新打开该会话
3. 上滑到顶部并触发“加载更早消息”

预期：
- 初始进入时优先显示最近消息
- 点击后能加载更早历史
- 不出现历史被覆盖、顺序错乱、重复消息

### M-05 离线队列

步骤：
1. 进入聊天页
2. 断网
3. 发送 2 到 3 条消息
4. 恢复网络

预期：
- 顶部出现离线提示条
- 离线发送不会崩溃
- 恢复网络后消息按顺序自动补发
- 不出现漏发或重复发

### M-06 桌面发现条

步骤：
1. 桌面端登录同一账号并在线
2. 打开移动端聊天页

预期：
- 出现桌面已发现条
- 显示桌面设备信息

### M-07 移动端发起 handoff

步骤：
1. 在手机端聊到半途中
2. 点击桌面发现条

预期：
- 提示 handoff 发送成功
- 桌面端自动打开聊天面板
- 桌面端恢复最近上下文，而不是空白新会话

### M-08 桌面剪贴板卡片

步骤：
1. 在桌面端复制一段明显文本
2. 等待移动端聊天页刷新

预期：
- 移动端出现“桌面剪贴板可用”卡片
- 卡片展示最近文本预览

### M-09 桌面剪贴板插入

步骤：
1. 在出现卡片时点击 `插入`

预期：
- 桌面剪贴板内容进入移动端输入框
- 不覆盖已有内容，而是追加到输入框中

### M-10 桌面剪贴板复制到手机

步骤：
1. 在出现卡片时点击 `复制`
2. 到其他输入位置粘贴

预期：
- 手机系统剪贴板中是最新桌面文本
- 弹出成功提示

## 5. 桌面端手工测试步骤

### D-01 悬浮球

步骤：
1. 启动桌面端
2. 找到悬浮球
3. 单击打开聊天面板

预期：
- 悬浮球可见
- 单击可打开聊天面板

### D-02 移动端 handoff 接收

步骤：
1. 保持桌面端在线
2. 由移动端发起 handoff

预期：
- 桌面端自动打开聊天面板
- 自动建立新的 handoff 会话 tab
- 最近消息上下文可见

### D-03 Spotlight 通知联动

步骤：
1. 触发一条带“View in Spotlight”的任务通知
2. 点击通知动作

预期：
- Spotlight 被打开
- 通知结果文本被带入 Spotlight 输入框
- 不再出现空 Spotlight

### D-04 Spotlight 选中文本

步骤：
1. 在桌面其他应用中选中一段文本
2. 打开 Spotlight

预期：
- Spotlight 可带入选中文本作为上下文
- 不应破坏原系统剪贴板内容

### D-05 剪贴板监听稳定性

步骤：
1. 多次打开/关闭聊天面板
2. 多次复制不同内容

预期：
- 不会出现重复 clipboard 事件
- 不会出现同一内容被重复同步多次

### D-06 聊天面板恢复

步骤：
1. 打开聊天面板并进行对话
2. 关闭后再打开

预期：
- 状态能恢复
- 不出现明显白屏或上下文丢失

## 6. 后端专项验证

### B-01 desktop-sync heartbeat

步骤：
1. 桌面端在线并已认证
2. 触发或等待心跳发送
3. 观察后端日志与 API 状态

预期：
- `POST /desktop-sync/heartbeat` 不返回 `500`
- `desktop_device_presence` 能正常 upsert

### B-02 unified devices

步骤：
1. 保持桌面端在线
2. 调用 unified devices 相关接口

预期：
- 移动端可见在线桌面设备
- `deviceType/source` 信息正确

### B-03 handoff 生命周期

步骤：
1. 由移动端发起 handoff
2. 桌面端接受并恢复会话

预期：
- handoff 记录创建成功
- 桌面端接收到 request/accepted 事件
- 会话上下文完整

## 7. 发布阻断项

以下任意一项失败，禁止发布：
- `desktop-sync heartbeat` 仍返回 `500`
- 移动端离线补发丢消息
- 移动端 handoff 到桌面后仍为空会话
- 桌面通知打开 Spotlight 仍丢失 payload
- 移动端桌面剪贴板入口不出现或动作无效
- 桌面端 selected text 仍污染系统剪贴板
